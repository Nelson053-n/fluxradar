## 1. Что за проект (в одном абзаце)

**FluxRadar** — публичный self-hosted веб-сервис мониторинга нод Flux-сети (https://runonflux.com). Пользователь вводит адрес кошелька → видит сводку по всем своим нодам (баланс, доход, статусы, бенчмарки, IP, версии). Параллельно — Telegram-бот, который шлёт алерты по критичным событиям (нода офлайн, бенчмарк упал, поступление на кошелёк). Аналог https://fluxnode.app.runonflux.io, но со своим дизайном и хостингом у нас.

## 3. Набор алертов (только эти — больше не предлагать)

**Группа A — состояние нод:**
1. Нода ушла в офлайн
2. Нода вернулась в онлайн (парный к 1)
3. Упал бенчмарк
4. Бенчмарк восстановился (парный к 3)
5. Устарела версия FluxOS
6. Устарела версия Benchmark

**Группа B — кошелёк:**
7. Поступление на кошелёк (любая входящая транзакция)
8. Выплата получена (входящая от Flux coinbase / fluxnode payout)

**НЕ делать в MVP:** алерты по DoS-list, maintenance window, изменению ранга. Это всё спам или преждевременная оптимизация.

## 4. Команды Telegram-бота (финальный список)

| Команда | Описание |
|---|---|
| `/start` | Приветствие + инструкция |
| `/link <wallet>` | Привязать кошелёк (несколько на один tg_user_id допустимо) |
| `/unlink <wallet>` | Отвязать |
| `/wallets` | Список привязанных |
| `/status` | Сводка по всем кошелькам: число нод, онлайн/офлайн, баланс |
| `/nodes [wallet]` | Список нод (без аргумента — по всем) |
| `/settings` | Inline-меню тогглов для типов алертов |
| `/mute 1h\|6h\|24h` | Заглушить алерты |
| `/unmute` | Снять заглушку |
| `/help` | Список команд |

## 5. Дизайн — что важно сохранить

**Источник истины:** `mockup_dashboard.html`. Это **рабочий HTML**, не картинка. Открой в браузере, потом открой как код — там все CSS-переменные, классы, анимации.

**Концепция:** «Cosmic radar dashboard». Тёмный индиго фон + три радиальных свечения (синий/циан/фиолетовый) + сетка-grid с radial-маской. Стиль монитора в центре управления.

**Палитра (НЕ менять):**
```
--bg-base:      #0A0E1A
--bg-deep:      #060912
--flux-primary: #2B61D1   (основной синий)
--flux-cyan:    #4FD7E8   (акцент, "живой", success-glow)
--flux-glow:    #5B8DEF   (промежуточный синий)
--flux-purple:  #7B5BFF   (Stratus, премиум)
--success:      #2FD3A0
--warning:      #F5B847
--danger:       #FF6B6B
```

**Шрифты (НЕ менять):**
- Текст: **Manrope** (300–800)
- Цифры/IP/адреса: **JetBrains Mono** (всё что должно "ощущаться как код или данные")

**Логотип:** круглый радар с крутящейся развёрткой (`conic-gradient` + `radarSweep` 4s анимация) и двумя концентрическими кольцами. Реализация в мокапе — `.logo-mark::before/::after`. Сохранить эту метафору при адаптации в React.

**Структура главного экрана (НЕ менять порядок):**
1. Top-bar: лого слева, навигация-пилюля по центру, live-цена FLUX справа
2. Wallet header — после загрузки кошелька (avatar + masked address + тогглы Auto-refresh / Privacy)
3. Первый ряд карточек: Total Nodes / Tier Breakdown / Estimated Earnings (на 2 колонки)
4. Второй ряд: Wallet Balance / Fleet Uptime / Bench Passed / Hosted Apps
5. Spotlight: 3 карточки (Best Uptime / Highest Rank / Most Hosted)
6. Node Overview — таблица с фильтр-табами и поиском
7. Telegram CTA — карточка с радиальным свечением и кнопкой "Open @FluxRadarBot"
8. Footer

**Что можно менять свободно:**
- Конкретные значения метрик (это плейсхолдеры)
- Иконки SVG → на Lucide React при портировании
- Способ анимаций (CSS → Framer Motion если нужно)
- Текст копи

**Чего избегать любой ценой:**
- Фиолетово-розовый градиент (canva-look, AI-generic)
- Шрифт Inter (дефолт)
- Emoji в навигации и кнопках (можно только флаги стран в таблице)
- Слишком много графиков на дашборде — MVP без них

## 6. Имя проекта — почему FluxRadar

Рабочее имя — **FluxRadar**. Аргументы:
- Метафора «радар» точно описывает то, что делает сервис: сканирует ноды по кругу, ловит сбои, посылает сигналы тревоги
- Поддерживает дизайн-концепцию (логотип, glow, CTA)
- 9 символов, легко произносится
- Хорошо для бота: `@FluxRadarBot`

**Не занято (на момент планирования):** `fluxscope`, `nodepulse` — заняты; `fluxradar` — предварительно свободен. **Проверить перед регистрацией:**
- Домены `.io / .app / .dev / .xyz` через namecheap/regru
- Бот через @BotFather
- GitHub org `github.com/fluxradar`

Если занят — fallback в порядке убывания: `FluxRadarNet`, `Flux-Radar`, `FluxOnRadar`, `MyFluxRadar`.

## 11. Критерий «MVP готов»

См. ТЗ §13. Кратко:
1. Открываешь сайт → вводишь известный «жирный» кошелёк → видишь корректные данные (сравнить с оригиналом fluxnode.app, ±1 мин погрешности кэша)
2. Privacy Mode работает
3. `/nodes` таблица сортируется, фильтруется
4. В Telegram: `/start` → `/link <адрес>` → выключаешь одну ноду → в течение 2 мин получаешь алерт «офлайн». Включаешь обратно → получаешь «онлайн»
5. `/status` возвращает корректную сводку
6. 100 одновременных запросов без падения (нагрузочный тест через `oha` или `k6`)
7. Lighthouse Performance ≥ 85 на главной
8. `cargo clippy -- -D warnings` зелёный на всём workspace
9. README с локальным запуском, `.env.example` заполнен

## 12. Сводка приоритетов (чтобы не сбиться)

1. **Корректность данных** — то, что видит пользователь, должно соответствовать реальности Flux-сети
2. **Алерты в Telegram приходят вовремя** — задержка ≤2 мин от события до пуша
3. **Дизайн соответствует мокапу** — палитра, шрифты, структура, glow
4. **Сервис не падает под нагрузкой** — есть rate-limit, есть кэш, нагрузочный тест проходит
5. **Безопасность** — никаких приватных ключей, HTTPS only, валидация входа
6. **Разработческий опыт** — README поднимает локально за 10 минут, clippy зелёный

Производительность Rust и крутость дизайна — приятный бонус, но **№1 всегда корректность данных**.
